# 思维导图 + 可拖拽任务画布（MVP）

技术栈：React 18+、TypeScript、Vite、Tailwind CSS v4、shadcn/ui 风格组件（Radix）、`react-konva`（深蓝网格背景）、`@xyflow/react`（无限画布、连线、缩放）。

## 启动

```bash
cd mindmap-canvas-mvp
npm install
npm run dev
```

浏览器打开终端里提示的本地地址（一般为 `http://localhost:5173`）。

生产构建：

```bash
npm run build
npm run preview
```

## 验收清单

- [ ] 左侧固定宽度约 260px，深蓝主题；根节点为「工作台」，其下为「项目」，项目下为「任务」三层结构。
- [ ] 项目可折叠/展开；可新增项目、删除项目；项目下可新增任务、删除任务。
- [ ] 点击左侧任务行：该项在脑图高亮，右侧画布对应节点选中并自动居中（轻微动画）。
- [ ] 右侧为可平移、滚轮缩放的画布；任务为可拖拽节点；节点两侧有连线桩，可在节点间拖拽连线。
- [ ] 双击节点打开对话框，可编辑标题、截止日期、优先级、状态；保存后左侧列表与节点展示同步更新。
- [ ] 在画布上选中/取消选中节点时，左侧任务高亮与 `selectedTaskId` 一致。
- [ ] 按 Delete / Backspace 删除画布选中节点时，对应任务从树与连线中移除。
- [ ] 刷新页面后，树结构、节点位置、连线、选中项（若仍存在）从 `localStorage`（键名 `mindmap-canvas-mvp-v1`）恢复。
- [ ] 无后端、无登录、无协作相关入口。

## 说明

- Konva 层用于与视口同步的网格背景；节点与边由 React Flow 渲染。
- 数据模型为「工作台 → 项目 → 任务」嵌套；画布仅展示「任务」节点，任务 `id` 与节点 `id` 一致以便联动。
